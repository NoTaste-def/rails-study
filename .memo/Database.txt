관계형 데이터베이스 (RDBMS):
- 데이터를 테이블 형태로 저장, 테이블 간의 관계를 정의하려 데이터를 관리함.
- SQL(Structed Query Language)을 사용하여 데이터를 검색하고 조작함.
- ex: MySQL, PostreSQL, Oracle, SQL Server, SQLite
- 특징: 데이터의 일관성과 정확성을 중시, 복잡한 데이터 관계를 표현하는 데 적합함.

NoSQL:
- 관계형 데이터베이스와 다른 방식으로 데이터를 저장하고 관리
- 다양한 데이터 모델 지원 (ex: key-value, 문서, 그래프 등)
- 분석 환경에서 확장성이 뛰어남.
- ex: MongoDB, Redis, Cassandra
- 특징: 유연성이 높고, 대용량 데이터 처리에 적합함.

[Rails에서의 데이터베이스]
주로 RDBMS를 사용하며, 데이터베이스를 쉽게 다룰 수 있도록 Active Record라는 ORM(Object-Relational Mapper)를 제공함.

- Active Record: 데이터베이스 테이블을 Ruby 객체로 매핑하여, SQL 코드를 직접 작성하지 않고도 데이터를 처리할 수 있도록 도와줌.
- SQLite: Rails 프로젝트를 처음 만들 때 기본적으로 설정되는 데이터베이스. 개발 환경에서 사용하기에 적합. (파일 기반 데이터베이스)
- MySQL, PostgreSQL: 실제 서비스 환경에서 많이 사용되는 데이터베이스.
- 데이터베이스 연결: config/database.yml 파일에서 데이터베이스 연결 설정을 함.

[데이터베이스 용어]
- Table: 데이터를 저장하는 기본 단위. 행(row)과 열(column)으로 구성되어 있음.
- Row / Record: 테이블에 저장된 데이터의 각 항목. (예: 사용자 정보, 게시글 정보) => 실제 데이터
- Column / Field: 테이블에 저장된 데이터의 속성 (예: 사용자 이름, 게시글 제목) => 데이터의 속성
- SQL: 데이터 베이스를 조작하기 위한 언어 (예: SELECT, INSERT, UPDATE, DELETE)
- Schema: 데이터베이스의 구조를 정의하는 것. (테이블 이름, 열 이름, 데이터 형식 등)
- Migration: 데이터베이스 스키마를 변경하는 방법. Rails에서는 마이그레이션 파일을 사용하여 스키마를 변경하고 관리함.
- ORM(Object-Relational Mapper): 데이터베이스 테이블을 객체로 매핑하여 데이터를 더 쉽게 조작할 수 있도록 도와주는 기술.

[데이터베이스 테이블 구조]
데이터베이스 테이블 구조는 테이블이 어떻게 구성되어 있는지, 
즉 테이블에 어떤 열(Column, Field)들이 들어있고, 각 열이 어떤 데이터 형식을 가지는지, 그리고 어떤 제약 조건이 있는지 등을 정의하는 것.

(Rails에서 데이터베이스 테이블 구조를 확인하는 방법)

1. 마이그레이션 파일 확인:
- 모델을 생성할 때 자동으로 생성되는 마이그레이션 파일은 테이블 구조를 정의하는 코드를 포함함.
- 마이그레이션 파일 위치 => db/migrate
- ex: db/migrate/YYYYMMDDHHMMSS_create_posts.rb

     class CreatePosts < ActiveRecord::Migration[8.0]
       def change
         create_table :posts do |t|
           t.string :title
           t.text :content

           t.timestamps
         end
       end
     end

 *   `create_table :posts`는 `posts` 테이블을 생성하는 명령어.
 *   `t.string :title`은 `title` 열을 문자열(`string`) 형식으로 생성.
 *   `t.text :content`는 `content` 열을 긴 문자열(`text`) 형식으로 생성.
 *   `t.timestamps`는 `created_at`과 `updated_at` 열을 자동으로 생성.

- 마이그레이션 파일을 통해 테이블의 열 이름, 데이터 형식, 제약 조건 등을 정확하게 확인 가능함.
- 가장 정확하고, 코드 기반으로 테이블 구조를 확인할 수 있는 방법.


2. Rails 콘솔에서 테이블 정보 확인:
- rails 콘솔을 사용하여 데이터베이스 테이블에 대한 정보 확인 가능
- 터미널에서 rails c / rails console 입력 => (객체이름).columns => 각 열(Column, Field) 마다 상세한 정보 확인 가능
 *   `Post.columns.map {|c| [c.name, c.sql_type]}` 와 같이 출력 형식을 변경하여 더 간단하게 볼 수도 있음.
 *   ex) `[["id", "INTEGER"], ["title", "varchar"], ["content", "TEXT"], ["created_at", "datetime(6)"], ["updated_at", "datetime(6)"]]`


3. DB Browser for SQLite 와 같은 데이터베이스 관리 도구 사용
- DB Browser for SQLite는 데이터베이스 파일을 시각적으로 확인할 수 있는 데이터베이스 관리 도구. (다운받으면 됨.)
- GUI 기반으로 테이블 구조를 확인할 수 있는 방법.
